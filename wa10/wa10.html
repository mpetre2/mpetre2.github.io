My extension builds on the existing random trivia generator by connecting it to the Useless Facts API. This API delivers randomized and funny bits of information, and also includes the source (the link) of where it comes from. Along with that there is an option to directly go to the app “X” (which was originally twitter), where users can directly make a post. The goal here is to get them to have a direct outlet to post the useless fact that they just generated.

I could imagine using an API in my critique project by using it to generate posts that appear on my redesigned Explore page. Rather than using a real social media feed, the API could pull random images or videos from themed datasets that match the topics a user selects (for example, “art,” “travel,” or “food”). This would simulate how Instagram recommends content, but in my design the user actively chooses the categories instead of the system deciding for them. Along with that this would randomize the users posts that are being presented. This helps everyone have an equal chance for their post to be displayed. Using the API this way would help demonstrate how giving users control over what data drives their feed creates a fairer, more transparent experience.
